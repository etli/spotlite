> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix four UI issues: Liked Songs card alignment, search results not closing on navigation, artist follower count in search, and Artists tab 403 scope error.

### Task 3: Close search results when navigating to album/artist

Fix: pass an `onNavigate` callback from `PanelShell` into `SearchResults`; call it when an `AlbumCard` link is clicked via an `onClick` prop on `AlbumCard`.

**Files:**
- Modify: `src/components/AlbumCard.tsx` — add optional `onClick?: () => void` prop, attach to the `<Link>` element
- Modify: `src/views/SearchResults.tsx` — accept `onNavigate?: () => void`, pass to album/artist `AlbumCard`s
- Modify: `src/components/PanelShell.tsx` — pass `onNavigate={() => setSearchQuery("")}` to `SearchResults`
- Modify: `src/views/__tests__/SearchResults.test.tsx`

### Task 4: Fix Artists tab 403 — add user-follow-read OAuth scope

The `/v1/me/following` endpoint requires the `user-follow-read` scope. It's missing from `SCOPES` in `spotify-auth.ts`. Adding it requires the user to re-authenticate (clear tokens and log in again) because existing tokens won't have the new scope.

**Files:**
- Modify: `src/lib/spotify-auth.ts:1-10`

- [ ] **Step 1: Add user-follow-read to SCOPES**

In `src/lib/spotify-auth.ts`, update the SCOPES array:

```ts
export const SCOPES = [
  "user-read-playback-state",
  "user-modify-playback-state",
  "user-read-currently-playing",
  "user-library-read",
  "user-follow-read",
  "playlist-read-private",
  "streaming",
  "user-read-private",
  "user-read-email",
];
```

- [ ] **Step 4: Note for user — re-auth required**

After deploying, the user must log out and log back in for the new scope to take effect. Existing sessions will still get 403 until tokens are refreshed with the new scope grant.
